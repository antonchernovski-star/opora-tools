# Opora Tools — Инструменты Опоры

Локальное приложение Bitrix24 для карточки сделки: показывает контакт клиента и даёт быстрые действия — WhatsApp, Telegram, Email, копирование телефона и email.

Статическое приложение без сборщиков: HTML5 + CSS3 + Vanilla JS + Bitrix24 JS SDK. Работает через GitHub Pages.

**Демо (вне Bitrix24 откроется в демо-режиме):**
https://antonchernovski-star.github.io/opora-tools/

---

## Возможности

- Автоматически определяет сделку, из карточки которой открыто приложение (`PLACEMENT` / `ENTITY_ID`).
- Загружает контакт сделки через REST: `crm.deal.get` → `crm.contact.get`.
- Показывает имя, фамилию, телефон и email контакта.
- Кнопка **WhatsApp** — очищает номер до цифр, меняет ведущую `8` на `7`, открывает `https://wa.me/НОМЕР`.
- Кнопка **Telegram** — открывает поиск пользователя по номеру `https://t.me/+НОМЕР`.
- Кнопка **Email** — открывает почтовый клиент через `mailto:`.
- Кнопки **Копировать телефон / email** — через `navigator.clipboard` (с фолбэком для iframe).
- Демо-режим при открытии вне Bitrix24 — удобно проверять интерфейс.

## Установка в Bitrix24

### 1. Опубликовать приложение (GitHub Pages)

Если разворачиваете свою копию:

1. Форкните / склонируйте репозиторий `opora-tools`.
2. В репозитории откройте **Settings → Pages**.
3. В блоке **Build and deployment** выберите:
   - Source: `Deploy from a branch`;
   - Branch: `main`, папка `/ (root)`.
4. Сохраните. Через 1–2 минуты приложение будет доступно по адресу
   `https://ВАШ_ЛОГИН.github.io/opora-tools/`.

### 2. Добавить локальное приложение в Bitrix24

1. В портале Bitrix24 откройте **Разработчикам** (Маркет → Разработчикам).
2. Выберите **Другое → Локальное приложение**.
3. Тип: **Серверное** (или «статичное» — без серверной части), галочка **«Использует только API»** должна быть СНЯТА (у приложения есть интерфейс).
4. Заполните поля:
   - **Ваш путь для обработчика**: `https://ВАШ_ЛОГИН.github.io/opora-tools/index.html`
   - **Путь для первоначальной установки**: тот же адрес.
5. В разделе **Права доступа (scope)** добавьте: `crm`, `placement`.
6. Сохраните — Bitrix24 выдаст `client_id` / `client_secret` (для статического приложения они не используются, но сохраните их).

### 3. Встроить во вкладку карточки сделки

Вариант через интерфейс: при создании локального приложения в блоке **Привязка к разделам** выберите размещение **Карточка сделки (вкладка)** — `CRM_DEAL_DETAIL_TAB`.

Вариант через REST (если привязываете вручную, выполните один раз от имени приложения):

```js
BX24.callMethod('placement.bind', {
    PLACEMENT: 'CRM_DEAL_DETAIL_TAB',
    HANDLER: 'https://ВАШ_ЛОГИН.github.io/opora-tools/index.html',
    TITLE: 'Инструменты Опоры'
});
```

После этого в каждой сделке появится вкладка **«Инструменты Опоры»**.

### Необходимые права (scope)

| Право | Зачем |
|-------|-------|
| `crm` | чтение сделки (`crm.deal.get`) и контакта (`crm.contact.get`) |
| `placement` | встраивание во вкладку карточки сделки и получение `PLACEMENT` / `ENTITY_ID` |

## Как это работает

1. `index.html` подключает Bitrix24 JS SDK: `https://api.bitrix24.com/api/v1/`.
2. При открытии внутри Bitrix24 модуль `bitrix.js` вызывает `BX24.init()`, читает `BX24.placement.info()` — получает `PLACEMENT` (например, `CRM_DEAL_DETAIL_TAB`) и `ENTITY_ID` (ID сделки из `options.ID`).
3. `crm.js` вызывает `crm.deal.get`, берёт `CONTACT_ID`, затем `crm.contact.get` и нормализует имя, фамилию, телефон, email.
4. `app.js` отрисовывает карточку и вешает обработчики кнопок.
5. Вне Bitrix24 (SDK недоступен) приложение включает демо-режим с тестовым контактом.

## Как добавить новый инструмент

Архитектура рассчитана на расширение отдельными JS-модулями — GetContact, Глаз Бога, ФССП, ЕФРСБ, AI-помощник, скоринг клиента, проверка телефона и любые другие:

1. Создайте файл `js/имя-инструмента.js` с модулем в пространстве имён `Opora` (шаблон — `js/whatsapp.js`):

```js
'use strict';
window.Opora = window.Opora || {};

Opora.GetContact = (function () {
    /**
     * Проверяет телефон через GetContact.
     * @param {string} phone
     */
    function check(phone) {
        // логика инструмента
    }
    return { check: check };
})();
```

2. Подключите модуль в `index.html` **до** `app.js`:

```html
<script src="js/getcontact.js"></script>
```

3. Добавьте кнопку в `index.html` и обработчик в функции `bindActions()` в `app.js`.

Данные контакта модуль получает в едином формате (`id`, `firstName`, `lastName`, `fullName`, `phone`, `email`) — ничего в `bitrix.js` и `crm.js` менять не нужно.

## Локальная проверка

Просто откройте `index.html` в браузере — приложение запустится в демо-режиме с тестовым контактом, и можно проверить интерфейс и кнопки.
